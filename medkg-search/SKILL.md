# medkg-search: 医学知识库检索技能

## 系统概览

**medkg** 是基于八年制医学教材第3版（39本PDF）构建的本地向量知识库。
- **数据库**：`C:\Users\lucha\WorkBuddy\medkg\medkg.db`（SQLite，向量维度1024）
- **向量模型**：BAAI/bge-m3（本地缓存，首次加载约3秒）
- **知识块总量**：77,525块 / 39本全量
- **OCR引擎**：Tesseract v5.5.0（中文识别率约85%，关键医学术语可检索）

## 搜索命令执行方式

**重要**：Python脚本必须用 `cmd /c` 包装（PowerShell对`--`参数处理有误）：

```cmd
cmd /c "set HF_ENDPOINT=https://hf-mirror.com && python.exe路径 脚本路径 查询词 --top N"
```

示例（查甲状腺结节）：
```cmd
cmd /c "set HF_ENDPOINT=https://hf-mirror.com && C:\Users\lucha\.workbuddy\binaries\python\envs\medkg\Scripts\python.exe C:\Users\lucha\WorkBuddy\medkg\03_search.py 甲状腺结节 --top 3"
```

## 核心搜索流程

### Step 1：执行语义搜索

```cmd
cmd /c "set HF_ENDPOINT=https://hf-mirror.com && python.exe 03_search.py "查询词" --top N"
```

| 参数 | 说明 |
|------|------|
| `--top N` | 返回条数，默认5，建议5~8 |
| `--book 书名` | 限定特定书籍 |
| `--cat 学科` | 限定学科分类（如`肿瘤科``神经科`） |

**注意**：必须设置 `HF_ENDPOINT=https://hf-mirror.com`，否则模型下载超时。

### Step 2：解读并格式化结果

`03_search.py` 输出格式：
```
[1] 书名  [学科]  相似度: 0.xxx
   块 N/M
   --------------------------------------------------
   知识块内容（已去重的医学段落）
```

**处理步骤**：
1. 提取最相关的3~5个知识块
2. 标注每条的**教材来源**和**相似度**
3. 合并多个来源，去重
4. 特别标注：诊断标准、关键指标、治疗方案、预后信息

### Step 3：结合核保/理赔实务

将医学知识转化为以下实务框架之一：

**核保场景输出框架**：
| 项目 | 内容 |
|------|------|
| 疾病定义 | 临床定义、分期/分级标准 |
| 核心指标 | 关键检验/影像指标及正常值 |
| 对预期寿命/死亡率影响 | 基于教科书医学依据 |
| 重疾险保障关联 | 是否属于重疾保障范围 |
| 核保建议 | 需要的补充资料、典型结论参考 |

**理赔场景输出框架**：
| 项目 | 内容 |
|------|------|
| 疾病定义 | 重疾定义条款 vs 实际诊断 |
| ICD-11编码 | 疾病分类代码 |
| 诊断依据充分性 | 病理/影像/实验室证据 |
| 治疗方案 | 是否符合理赔条款要求 |
| 审核要点 | 既往症/等待期/如实告知 |

## 重要限制说明

1. **OCR局限**：知识库基于Tesseract OCR提取，中文医学术语识别率约85%，关键数值/术语可检索，但存在少量乱码
2. **知识时效**：教材代表经典医学知识，**不含**2025-2026最新指南更新（NCCN/ESC/ADA等最新版本需另行检索）
3. **条款优先**：返回的医学知识是参考依据，**最终承保/理赔结论**必须严格对照具体产品条款
4. **本地执行**：全程本地运行，无网络依赖，保护数据隐私

## 典型应用示例

### 示例1：核保问诊"肺结节"

用户：肺结节8mm，核保怎么考虑？

执行：
```

03_search.py 肺结节 核保 --top 2
```

输出框架：
| 项目 | 内容 |
|------|------|
| 影像学特征 | 恶性指标：毛刺征、分叶、胸膜牵拉、空洞 |
| 临床处理 | 8mm处理原则（随访/增强CT/PET/穿刺）|
| 核保参考 | 结合影像描述（毛刺/分叶→高危→延期/除外）|
